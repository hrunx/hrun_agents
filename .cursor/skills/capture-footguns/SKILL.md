# Capture Footguns

## When to use
- Use after debugging incidents or discovering “gotchas”.
- Use after adding new integrations, migrations, or deployment steps.

## Instructions
1) Read `.planning/RISKS.md`.
2) Add concise entries under Known risks or Past incidents.
3) Include: symptom, root cause, mitigation, and how to verify.
4) Do not include secret values or sensitive operational details.

## Output
- Risks added/updated
- Links to impacted files
